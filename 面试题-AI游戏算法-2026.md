## 一、AI / 大模型 (LLM) 高频面试题

### 1.1 基础概念与架构

#### 1. Transformer 架构的核心组件
- **Self-Attention 机制**：Query、Key、Value 的计算逻辑
- **Multi-Head Attention**：多头注意力的作用和实现
- **Positional Encoding**：位置编码的几种方式（正弦/可学习）
- **残差连接与 Layer Normalization**

#### 2. BERT vs GPT 的区别
| 维度 | BERT | GPT |
|------|------|-----|
| 架构 | Encoder-only | Decoder-only |
| 训练目标 | Masked LM + NSP | 自回归语言建模 |
| 适用场景 | 理解任务（分类、NER） | 生成任务（对话、写作） |
| 注意力 | 双向 | 单向（Causal） |

#### 3. Prefix LM vs Causal LM
- **Causal LM**：严格自回归，只能看到前面的 token
- **Prefix LM**：prefix 部分双向可见，生成部分自回归（如 T5、GLM）

#### 4. Tokenizer 原理
- **BPE** (Byte Pair Encoding)：GPT 系列使用
- **WordPiece**：BERT 使用
- **SentencePiece**：多语言支持，不依赖空格分词
- **Unigram**：T5 使用

#### 5. Layer Normalization 变体
- **Pre-LN vs Post-LN**：稳定性与收敛速度
- **RMSNorm**：去除均值中心化，LLaMA 使用
- **DyT (Dynamic Tanh)**：2025 新技术，替代 LayerNorm

---

### 1.2 进阶技术

#### 6. 大模型幻觉 (Hallucination) 解决方案
- **RAG**：检索增强生成
- **Fact-checking**：事实核查模块
- **Chain-of-Verification**：自我验证链
- **训练数据清洗**：减少噪声数据
- **RLHF**：人类反馈对齐

#### 7. RAG (Retrieval-Augmented Generation) 架构
```
用户查询 → 向量化 → 向量检索 → 召回相关文档 → 拼接 Prompt → LLM 生成
```
- **向量数据库**：Milvus、Pinecone、Weaviate
- **Embedding 模型**：text-embedding-ada-003、BGE、M3E
- **重排序 (Rerank)**：Cross-encoder 精排
- **查询改写**：HyDE、Query Expansion

#### 8. 长上下文处理技术
- **Ring Attention**：环形注意力，突破显存限制
- **LongRoPE**：位置编码外推
- **Sliding Window Attention**：滑动窗口注意力
- **稀疏注意力**：BigBird、Longformer

#### 9. 模型微调方法
| 方法 | 原理 | 显存占用 | 适用场景 |
|------|------|----------|----------|
| Full Fine-tuning | 全参数更新 | 高 | 数据充足 |
| LoRA | 低秩适配器 | 低 | 通用微调 |
| QLoRA | 4-bit 量化 + LoRA | 极低 | 消费级显卡 |
| Adapter | 插入小型适配层 | 低 | 多任务 |
| Prompt Tuning | 软提示学习 | 极低 | 大模型冻结 |
| P-Tuning v2 | 深层提示微调 | 低 | NLU 任务 |

#### 10. 推理优化技术
- **KV Cache**：缓存键值对，避免重复计算
- **量化**：INT8、INT4、GPTQ、AWQ
- **Speculative Decoding**：投机采样，小模型草稿+大模型验证
- **Continuous Batching**：动态批处理
- **PageAttention**：vLLM 的内存管理

#### 11. Agent 框架
- **ReAct**：推理+行动交替
- **Tool Use / Function Calling**：工具调用
- **Plan-and-Solve**：先规划后执行
- **Multi-Agent**：多智能体协作

#### 12. 对齐技术
- **RLHF**：人类反馈强化学习（PPO）
- **DPO**：直接偏好优化，无需奖励模型
- **RLAIF**：AI 反馈强化学习

---

### 1.3 2025-2026 新技术热点

#### DyT (Dynamic Tanh) 替代 LayerNorm
- 背景：LayerNorm 计算开销大
- 方案：用可学习的 tanh 缩放替代
- 效果：训练更快，性能持平

#### Gemini 2.5 Pro 思考推理范式
- 原生支持多模态推理
- 长上下文（100万 token）
- 思维链可视化

#### 多模态 Agent
- 统一处理文本、图像、音频、视频
- 视觉-语言模型：GPT-4V、Claude 3、Gemini
- 视频理解：Video-LLaMA、VideoChat

## 二、游戏 + AI 面试题

### 2.1 游戏 AI 基础

#### 1. 游戏 AI 架构对比
| 架构 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| FSM (有限状态机) | 简单直观 | 状态爆炸 | 简单 NPC |
| Behavior Tree | 模块化、可复用 | 设计复杂 | 复杂 AI |
| GOAP (目标导向) | 灵活、可扩展 | 计算量大 | 策略游戏 |
| Utility AI | 多因素决策 | 权重调参难 | 复杂决策 |

#### 2. 寻路算法
- **A***：启发式搜索，最常用
- **Dijkstra**：无权图最短路径
- **NavMesh**：导航网格，3D 游戏标准
- **Flow Field**：群体寻路，RTS 游戏
- **JPS (Jump Point Search)**：网格图优化

#### 3. 游戏决策算法
- **Minimax + Alpha-Beta 剪枝**：棋类 AI
- **MCTS (蒙特卡洛树搜索)**：围棋、复杂博弈
- **行为树节点类型**：Sequence、Selector、Parallel、Decorator

#### 4. 强化学习在游戏中的应用
- **DQN**：Atari 游戏
- **PPO**：稳定策略梯度，OpenAI Five
- **模仿学习**：从人类演示学习
- **AlphaStar / OpenAI Five**：多智能体强化学习

## 三、算法基础（手撕代码必考）

### 3.1 数据结构
1. **链表**：反转、合并、环检测
2. **树**：遍历（前/中/后/层序）、LCA、序列化
3. **图**：DFS、BFS、拓扑排序、最短路径
4. **堆**：Top K、合并 K 个有序数组
5. **并查集**：连通性、最小生成树 (Kruskal)

### 3.2 算法
1. **动态规划**：背包、最长子序列、编辑距离
2. **贪心**：区间调度、 Huffman 编码
3. **回溯**：全排列、N 皇后、子集
4. **双指针/滑动窗口**：子串问题
5. **二分查找**：旋转数组、找边界

### 3.3 机器学习基础
1. **梯度下降**：BGD、SGD、Adam
2. **过拟合/欠拟合**：原因与解决方案
3. **交叉验证**：K-Fold、留一法
4. **评估指标**：准确率、精确率、召回率、F1、AUC

### 3.4 深度学习
1. **CNN**：卷积、池化、ResNet、BatchNorm
2. **RNN/LSTM/GRU**：时序建模
3. **Attention**：Self-Attention、Cross-Attention
4. **正则化**：Dropout、Weight Decay
