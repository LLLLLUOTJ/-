### 三、专业英语问题

#### 计算机基础
23. **What is the difference between process and thread?**（进程和线程的区别）

进程是资源分配的基本单位, 线程是CPU调度的基本单位。进程之间相互独立, 线程之间共享进程的资源。进程通信需要IPC机制, 线程通信更简单。进程切换开销较大, 线程切换开销较小。

translate: A process is the basic unit of resource allocation, while a thread is the basic unit of CPU scheduling. Processes are independent of each other, while threads share the resources of their process. Inter-process communication requires IPC mechanisms, while thread communication is simpler. Process switching has a higher overhead compared to thread switching.

24. **Explain the TCP/IP model**（解释TCP/IP模型）

TCP/IP模型分为四层：应用层、传输层、网络层和数据链路层。应用层负责处理特定的应用协议，如HTTP、FTP等；传输层提供端到端的通信服务，主要协议有TCP和UDP；网络层负责数据包的路由和转发，主要协议是IP；数据链路层负责在物理介质上进行数据传输，涉及MAC地址和帧结构。

translate: The TCP/IP model consists of four layers: the Application Layer, Transport Layer, Network Layer, and Data Link Layer. The Application Layer handles specific application protocols such as HTTP and FTP; the Transport Layer provides end-to-end communication services, with main protocols being TCP and UDP; the Network Layer is responsible for routing and forwarding packets, with the main protocol being IP; the Data Link Layer is responsible for data transmission over physical media, involving MAC addresses and frame structures.

25. **What is virtual memory?**（什么是虚拟内存？）

虚拟内存是一种内存管理技术，它使得程序可以使用比实际物理内存更大的地址空间。通过页表将虚拟地址映射到物理地址，虚拟内存允许程序在需要时将部分数据从物理内存交换到磁盘上，从而实现更高效的内存利用和更大的程序运行空间。

translate: Virtual memory is a memory management technique that allows programs to use a larger address space than the actual physical memory. By using page tables to map virtual addresses to physical addresses, virtual memory enables programs to swap parts of data from physical memory to disk when needed, thus achieving more efficient memory utilization and a larger runtime space for programs.

26. **Explain the difference between stack and queue**（栈和队列的区别）

栈是一种后进先出（LIFO）的数据结构，元素只能在栈顶进行插入和删除。队列是一种先进先出（FIFO）的数据结构，元素只能在队尾插入，在队头删除。栈适用于递归、表达式求值等场景，队列适用于BFS、任务调度等场景。

translate: A stack is a Last-In-First-Out (LIFO) data structure where elements can only be inserted and removed from the top of the stack. A queue is a First-In-First-Out (FIFO) data structure where elements can only be inserted at the rear and removed from the front. Stacks are suitable for scenarios like recursion and expression evaluation, while queues are suitable for scenarios like BFS and task scheduling.

27. **What is object-oriented programming?**（什么是面向对象编程？）

面向对象编程是一种编程范式，它将数据和操作数据的函数封装在一起，形成一个个对象。每个对象都有属性（数据）和方法（函数），通过类来定义对象的结构和行为。面向对象编程强调封装、继承和多态等特性，使得代码更易于维护和扩展。

translate: Object-oriented programming is a programming paradigm that encapsulates data and the functions that operate on that data together into objects. Each object has attributes (data) and methods (functions), and classes are used to define the structure and behavior of objects. Object-oriented programming emphasizes features such as encapsulation, inheritance, and polymorphism, making code easier to maintain and extend.

#### 数据结构
28. **Explain binary search tree**（解释二叉搜索树）

二叉搜索树是一种特殊的二叉树，其中每个节点的值都大于其左子树中所有节点的值，并且小于其右子树中所有节点的值。这种结构使得在平均情况下，搜索、插入和删除操作的时间复杂度为O(logn)，但在最坏情况下（如退化成链表）可能达到O(n)。

translate: A binary search tree is a special type of binary tree where the value of each node is greater than the values of all nodes in its left subtree and less than the values of all nodes in its right subtree. This structure allows for average-case time complexity of O(logn) for search, insertion, and deletion operations, but in the worst case (such as when it degenerates into a linked list), it can reach O(n).

29. **What is the time complexity of quicksort?**（快排的时间复杂度）

平均时间复杂度是O(nlogn)，最坏时间复杂度是O(n²)，空间复杂度是O(logn）。

translate: The average time complexity of quicksort is O(nlogn), the worst-case time complexity is O(n²), and the space complexity is O(logn).

30. **Explain dynamic programming**（解释动态规划）

动态规划是一种算法设计方法，适用于具有重叠子问题和最优子结构性质的问题。它通过将原问题分解为更小的子问题，并将子问题的解存储在一个表格中，以避免重复计算，从而提高效率。总之就是通过记忆化递归或者迭代的方式来避免重复计算，提升效率。

translate: Dynamic programming is an algorithm design method suitable for problems with overlapping subproblems and optimal substructure properties. It breaks down the original problem into smaller subproblems and stores the solutions to these subproblems in a table to avoid redundant calculations, thus improving efficiency. In summary, it uses memoization or iterative approaches to avoid redundant calculations and enhance efficiency.

#### 算法
31. **Explain Dijkstra's algorithm**（解释Dijkstra算法）

Dijkstra算法是一种用于求解单源最短路径问题的贪心算法。它通过维护一个优先队列来不断选择当前距离源点最近的未访问节点，并更新其邻居节点的距离，直到所有节点都被访问过。该算法适用于图中边权非负的情况。

translate: Dijkstra's algorithm is a greedy algorithm used to solve the single-source shortest path problem. It maintains a priority queue to continuously select the unvisited node closest to the source and updates the distances of its neighboring nodes until all nodes have been visited. This algorithm is suitable for graphs with non-negative edge weights.

32. **What is the difference between BFS and DFS?**（BFS和DFS的区别）

BFS是一种按广度遍历的方法, 适用于寻找最短路径等问题；DFS是一种按深度遍历的方法, 适用于搜索、回溯等问题。BFS使用队列实现, DFS使用栈或递归实现。

translate: BFS is a breadth-first traversal method, suitable for problems like finding the shortest path; DFS is a depth-first traversal method, suitable for search and backtracking problems. BFS is implemented using a queue, while DFS is implemented using a stack or recursion.

## 第四部分：面试技巧与注意事项

### 一、答题技巧

#### STAR法则
- **S**ituation（情境）：背景介绍
- **T**ask（任务）：要解决的问题
- **A**ction（行动）：采取的措施
- **R**esult（结果）：取得的成果

#### 不会的问题怎么办？
1. **诚实承认**："老师，这个问题我不是很确定"
2. **相关知识点**："但我了解相关的XXX"
3. **学习意愿**："面试后我会去深入学习"
4. **不要瞎编**：不懂装懂是大忌

---

### 二、注意事项

#### 着装礼仪
- 商务休闲装，干净整洁
- 不要穿得太随意或太正式

#### 言行举止
- 进门敲门，得到允许后再进入
- 主动问好，保持微笑
- 坐姿端正，不要抖腿
- 眼神交流，不要东张西望
- 离场时致谢

#### 语言表达
- 语速适中，不要太快或太慢
- 声音洪亮，自信大方
- 逻辑清晰，分点回答
- 避免口头禅（"嗯"、"啊"、"然后"）
